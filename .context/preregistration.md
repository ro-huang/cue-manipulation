# Pre-registration: Stated vs grounded confidence under Koriat-style cue manipulations

**Date filed:** 2026-04-26.
**Status:** Llama-3.1-8B-Instruct data already collected (`runs/mve_llama8b/`); analysis on that data is **exploratory by definition** and is reported as such. The Qwen-2.5-7B replication and any subsequent model is **confirmatory** under this document.

This pre-registration locks the design and primary analysis. It supersedes any earlier informal description and is the document a future co-author or reviewer should read first. Any deviation must be flagged explicitly in the writeup as "exploratory."

---

## 1. Headline claim (the thing being tested)

> Under Koriat-style cue manipulations of LLM trivia answering, **stated** confidence channels (first-token log-probability of the answer; verbalized categorical rating) inflate even when accuracy is flat or drops; **grounded** confidence channels (Kadavath P(True) self-elicitation) do not show the same inflation. The dissociation generalizes across model families.

Falsifiable: if `p_true` rises with the same effect-size ordering as `log_prob` and `verbal_cat` across the FOK conditions, OR if neither channel shows inflation, OR if the pattern reverses, the claim fails.

---

## 2. Locked design

### 2.1 Items
The 461 TriviaQA items in `runs/mve_llama8b/items_filtered.jsonl` (single-entity answers, Llama-3.1-8B baseline accuracy in [0.2, 0.8], passed the prime-leak validator at the original 5% threshold). The same item set is reused across all replication models without re-filtering. Per-model accuracy distributions will differ; the dissociation claim is about *confidence-vs-accuracy patterns within model*, not absolute accuracy.

### 2.2 Primes
The same prime text per (item, condition) generated by `claude-opus-4-7` is reused across replication models (`runs/mve_llama8b/primes.jsonl`). Primes are model-independent text; reusing them keeps the manipulation identical across replication models.

### 2.3 Conditions (run set)

All eight existing conditions are run for completeness, but the primary analysis uses six. The primary set:

| Condition | Role | Predicted Δ stated conf | Predicted Δ grounded conf | Predicted Δ accuracy |
|---|---|---|---|---|
| `baseline` | reference | 0 | 0 | 0 |
| `cue_familiarity_priming` | FOK manipulation | ↑ | ≈0 | ≈0 (must not rise — leak control) |
| `partial_accessibility` | FOK + misleading | ↑ | ≈0 or ↓ | ↓ |
| `target_priming` | positive control | ↑ (legitimate) | ↑ | ↑ |
| `random_paragraph` | yoked irrelevant content | ≈0 to ↓ | ≈0 to ↓ | ≈0 to ↓ |
| `whitespace` | length-only artifact | ≈0 | ≈0 | ≈0 |

Secondary (run but not load-bearing on the headline): `illusory_tot`, `fluency_degradation`. Reported separately in a short "secondary findings" section.

### 2.4 Confidence measures

| Measure | Channel | Role |
|---|---|---|
| `log_prob` | stated (model behavior) | **Primary stated** — first-token log-probability of the answer. |
| `verbal_cat` | stated (introspection) | **Primary stated** — verbalized 5-point categorical rating. |
| `p_true` | grounded (Kadavath self-elicit) | **Primary grounded** — P(True) of "Is the answer correct? True/False" |

`verbal_num` is run for completeness but **not** part of the primary analysis (largely redundant with `verbal_cat`). `caa_proj` is **out of scope** for this branch — its activation-axis story is handled separately on the `feel-known-axis` branch per `.context/feel_known_axis_spec.md`. For replication models, `caa_proj` is not collected.

### 2.5 Models

| Tier | Model | Status |
|---|---|---|
| Discovery (exploratory) | Llama-3.1-8B-Instruct | Already collected (`runs/mve_llama8b/`) |
| Replication (confirmatory) | Qwen-2.5-7B-Instruct | To run (this branch) |
| Replication (confirmatory, optional) | Mistral-7B-Instruct-v0.3 | Optional second replication |

Adding a same-family scale-up (Llama-3.1-70B) is explicitly **not** part of the headline test — same family is the least informative direction for the generality question.

### 2.6 Sample size justification

n=461 items × 6 primary conditions × 3 primary measures = 8298 trial-measure observations per model. Paired tests on item_id give n=461 paired observations per (condition, measure). The existing 8B run shows the smallest primary effect (`p_true` shift in `cue_familiarity_priming`) at |Δ| ≈ 0.01 (n.s.) and the largest (`log_prob` shift in `partial_accessibility`) at |Δ| ≈ 0.13. A paired n of 461 with σ ≈ 0.18 is overpowered for any |Δ| > 0.04 (α=.05, power 0.95). Adequate.

## 3. Primary analysis

For each model, for each of the five non-baseline primary conditions, for each of the three primary confidence measures and for accuracy:

1. Compute the **paired Δ from baseline** per item: `Δ_i = measure(primed_i) − measure(baseline_i)` (and similarly for accuracy).
2. Report mean Δ, paired 95% CI (1000-draw bootstrap), and paired t-test p-value.
3. Report a clustered OLS regression (cluster on `item_id`) of Δ-measure on condition fixed effects and the Δ-accuracy×condition interaction. Reference category = `baseline` (per pre-reg; differs from the existing `regression_clustered.txt` which uses `cue_familiarity_priming` as reference and is to be re-run).

Headline figure: `runs/<run-name>/plots/headline.png` — produced by `scripts/10_headline_figure.py`. One row per model. Within each row, three colored bars per condition (the three primary measures) + one bar for accuracy, with paired CIs and significance stars. The figure is the primary visual artifact.

### 3.1 Pass/fail criteria

Primary criterion (the claim, ANDed across all replication models):

- **C1.** `cue_familiarity_priming`: paired Δ on `log_prob` > 0 at p < .05, paired Δ on `verbal_cat` > 0 at p < .05, paired Δ on `p_true` not significantly different from 0 (p > .05 or |Δ| < 0.02), paired Δ on accuracy not significantly different from 0 (p > .05 or |Δ| < 0.03).
- **C2.** `partial_accessibility`: paired Δ on `log_prob` > 0 at p < .05, paired Δ on `verbal_cat` > 0 at p < .05, paired Δ on `p_true` either flat (p > .05) or smaller in magnitude than `log_prob`, paired Δ on accuracy < 0 at p < .05.
- **C3.** `target_priming`: paired Δ on accuracy > 0 at p < .05 (sanity check that the positive control works).

The headline claim **passes** for a given model iff C1 ∧ C2 ∧ C3 hold.

The headline claim **passes overall** iff it passes on Llama-3.1-8B-Instruct (exploratory) **and** at least one replication model (confirmatory). If only the 8B passes, the result is "Llama-specific" and reported as such.

### 3.2 Secondary analyses (reported, not pre-registered as confirmatory)

- `illusory_tot`: pattern of Δ on each measure. Predicted by Schwartz 1998 to behave like `cue_familiarity_priming`; if null in this model, report as null finding.
- `fluency_degradation`: pattern of Δ on each measure. Predicted by Whittlesea-style fluency to lower confidence; the existing 8B data show the opposite. Report observed pattern across models.
- `random_paragraph`: report as a non-trivial result, not a null artifact control. The 8B verbal-confidence depression here is itself a finding ("LLMs hedge in the presence of irrelevant context").
- `verbal_num` results, condition-by-condition.
- Cross-measure item-level correlation matrix.
- Differential effect of order counterbalance (if any model is run with `counterbalance_order: true`).

## 4. What is deliberately out of scope here

- Activation-space analyses (CAA, DAS, steering, feel-known axis) — handled on `feel-known-axis`.
- Model size (8B → 70B in same family).
- New conditions or new measures beyond what's listed.
- Re-generating primes (the existing primes are reused across models).
- Re-tuning the leak validator (a separate sub-task; if it's tightened, it produces a new item set with its own pre-reg).

## 5. Deviations log

Any deviation from this document must be appended here with a date, what changed, and why. Deviations made post-hoc invalidate the confirmatory status of the affected analyses; they remain reportable as exploratory.

### 2026-04-26 — Qwen-2.5-7B replication results

The Qwen replication did not pass the locked headline criteria:
- **C1 FAILS:** `cue_familiarity_priming` produces no significant lift on `log_prob` (Δ ≈ −0.004) or `verbal_cat` (Δ ≈ +0.009) on Qwen.
- **C2 PARTIAL:** `partial_accessibility` raises `log_prob` (+0.113, p < .001) and drops accuracy (−0.254, p < .001) — both replicate. But `verbal_cat` does not lift on Qwen (Δ ≈ +0.014, n.s.).
- **C3 PASSES:** `target_priming` raises accuracy (+0.377, p < .001).

**Implication.** Per pre-reg §3.1, "headline claim passes overall iff C1 ∧ C2 ∧ C3 hold on at least one replication model." Qwen does not pass C1, so the original headline (cue-familiarity dissociation) is not confirmed across model families. The robust cross-model finding is the partial-accessibility signature: log_prob inflation paired with accuracy collapse.

**Reframe (exploratory until further replication).** The replicating claim is narrower: *partial-accessibility primes induce a Koriat-style log-prob-confidence-vs-accuracy dissociation that generalizes across model families.* Cue-familiarity becomes a Llama-specific secondary finding. Verbal-channel responsiveness is also model-specific (`verbal_cat` is dead on Qwen for every condition).

Headline figure with both models: `runs/replication_qwen7b/plots/headline.png`.

### 2026-04-26 — Cross-model replication v2 (Llama-3.1-70B, Qwen-2.5-72B)

Extending replication to two larger models (same families as the discovery + first replication). Two operational deviations:

1. **Precision.** 70B/72B run in `int4` (bitsandbytes NF4 with bf16 compute), not `bfloat16`, to fit on a single H100 80GB. Initially attempted `int8` but it didn't reliably fit on 80GB at load time (~70GB nominal weights but transformers' 8-bit loader needed extra fp32 buffers and started CPU-offloading layers). int4 lands at ~43 GB used and runs comfortably. Discovery and replication-v1 used `bfloat16`. Effect direction is generally robust to inference precision for QA tasks at this scale (NF4 + bf16 compute preserves perplexity well per QLoRA literature); deviation noted to be re-evaluated if the 70B-class results are surprising.
2. **Gemma-2-{9B, 27B} dropped from v2.** Originally part of the v2 plan; user dropped them after license-acceptance discussion (2026-04-26). Replication v2 is therefore Llama-3.1-70B + Qwen-2.5-72B only. The config retains a stub comment so they can be added in a future run if desired (the resumable runner would fill those rows without re-running Llama/Qwen).

Output dir: `runs/replication_v2/` (single parquet, two model rows).
